NASA Network Sniffer Project (2020)

Problem statement

A significant amount of traffic passes between the Space Launch System and Mission Control every second. That traffic flow, though vital, had very little transparency; engineers couldn't tell how many packets were moving through the stream per second, the size of the packets, or how efficiently the rocket bundled data for release.

Our solution

I built a non-subscribing network sniffer that sat on top of the data stream and monitored the data throughput on the pipeline, the size of the packets, and the number of measurements transmitted per second. I then built a CLI and GUI to run the tool for Mission Control.
